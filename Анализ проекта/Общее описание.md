## Введение

Настоящее ТЗ описывает версию 1.0 (MVP). В рамках данной версии реализуется функционал создания среды для ведения ручного тестирования и автоматического тестирования. Добавление интеграций с нейросетями и создание пространства для разработчиков выходят за рамки данного документа и будут описаны в ТЗ на версию 2.0.
## Идея

Web-приложение для управления тестирования ПО (TMS) с функцией контроля и отслеживания процессов тестирования ПО для компаний стремящихся оптимизировать время тестирования и сократить денежные и временные затраты.

**Главное отличие  от конкурентов:**
- Низкая цена (относительно конкурентов)
- Возможность переноса данных из других ТМС
- Более удобный опыт взаимодействия
**Основные задачи:**
- Организация процесса тестирования
- Отслеживаемость процессов тестирования
- Аналитика и отчётность
## Цели

1. Конкурировать с TestIT в сегменте Русских TMS
2. Монетизация: Платные подписки за количество пользователей (до 5 -бесплатно для малых компаний)
## Аудитория

Целевой аудиторией являются бизнес в IT сфере, связанный с разработкой ПО и/или его тестированием.
В рамках бизнеса существуют 3 портрета пользователей TMS:
1. **QA-инженер** (Ручной).
*Потребность:* "Я хочу быстро создать тест-кейс по чек-листу и не заполнять 50 полей"
2. **Automation QA**.
*Потребность:* "Мне нужен API, чтобы привязать сюда Allure-отчёт и видеть, какие шаги упали"
3. **Менеджер/Тимлид**.
*Потребность:* "Я хочу видеть красивый Dashboard с графиком багов и тепловой картой покрытия по модулям"

## Роли

### Администратор
- Полный контроль над всеми разделами
- Назначение ролей
- Выдача и создания профилей для сотрудников
- Полный доступ к логам
### Team Lead
- Создание и архивация проектов
- Назначение  членов команды на прогоны
- Создание, удаление и редактирование Дэшбордов
### QA-инженер
-  Может создавать и редактировать тест-кейсы
-  Прохождение назначенных прогонов и изменение статусов шагов
-  Создание отчёта о дефектах
### API Bot
- Используется для токенов доступа CI/CD систем.
- Права: Создание нового тестового прогона и запись результатов автотестов. Запрещено: удаление данных и доступ к биллингу.
